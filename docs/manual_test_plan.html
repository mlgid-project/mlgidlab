Collecting and sending logs

There is no log file on disk — logs live in the app and the terminal. Three ways to capture them, easiest first:

  1. Copy diagnostics (quickest): Help → Copy diagnostics copies versions + active session + the last 50 log lines to your clipboard. Paste into your email. Nothing is uploaded.
  2. Full Logs dock: click inside the Logs dock, select all (Ctrl+A), copy (Ctrl+C), paste into a text file — for more than the last 50 lines.
  3. Terminal output (crashes/freezes): copy everything the terminal you launched mlgidlab from printed, especially any Traceback (most recent call last) block.

Then click Copy results to clipboard above, paste it into an email together with the logs, and send to nico.lerch@uni-tuebingen.de (or open an issue at github.com/mlgid-project/mlgidLAB/issues).
